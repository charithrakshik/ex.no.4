# Explanation

Prompt engineering helps in designing clear instructions for AI models to generate useful responses. Different prompt patterns help AI provide better structured answers depending on the situation.

In this experiment, prompts are designed for a manufacturing automation system using IoT and embedded controllers. The responses generated by AI tools such as ChatGPT and Gemini are compared and evaluated using a rubric method.

# Procedure
# 1. Define the Scenario and Use Case
Scenario

The manufacturing industry is trying to reduce manual supervision and increase production efficiency through automation. The system will use IoT sensors and embedded controllers to monitor equipment performance and automate operations.

The system can:

• Monitor machine conditions in real time
• Detect faults before breakdown occurs
• Provide predictive maintenance alerts
• Improve energy efficiency in manufacturing processes

This helps industries reduce downtime and improve productivity.

Target Audience

Manufacturing industries such as:

• Automotive industry
• Electronics manufacturing
• Food processing industries

These sectors can greatly benefit from automation and smart monitoring systems.

# Main Objectives

• Improve production efficiency by approximately 30%
• Reduce unexpected machinery failures
• Enable real-time monitoring of factory systems
• Reduce energy consumption through optimized machine usage

# 2. Identify Prompt Patterns for Each Design Aspect
# Comparative Analysis Prompt
# Definition

Comparative prompts ask AI to compare two or more technologies or concepts and explain their differences and advantages.

# Prompt

"Compare IoT-based predictive maintenance with traditional manual maintenance in a manufacturing factory."

# ChatGPT Output (Example)

IoT-based predictive maintenance uses sensors to monitor machine health continuously and predict failures before they occur. Traditional maintenance relies on periodic manual inspections and repairs after failures. Predictive maintenance reduces downtime and maintenance costs.

# Gemini Output (Example)

Predictive maintenance uses data from IoT sensors to detect early warning signs of equipment failure. Manual maintenance depends on scheduled checks or repairs after breakdown. Predictive systems improve efficiency and reduce operational disruptions.

# Experiential Perspective Prompt
# Definition

This prompt asks the AI to answer from the perspective of a particular role or professional experience.

# Prompt

"As a factory maintenance engineer, explain how IoT-based machine monitoring helps you manage equipment maintenance."

# ChatGPT Output

IoT monitoring allows engineers to track machine health in real time, receive alerts when abnormal patterns occur, and schedule maintenance before breakdown happens.

# Gemini Output

From an engineer’s perspective, IoT sensors simplify maintenance planning by providing performance data and automated alerts, helping prevent unexpected machine failures.

# Everyday Functioning Prompts
# Definition

Everyday functioning prompts explain technical ideas using simple real-life examples.

# Prompt

"Explain IoT predictive maintenance using an example similar to a health monitoring smartwatch."

# ChatGPT Output

Just like a smartwatch tracks heart rate and activity to detect health issues, IoT sensors monitor machine temperature, vibration, and performance to detect possible failures.

# Gemini Output

IoT monitoring works like a smartwatch that tracks health conditions. If unusual patterns are detected, alerts are sent so action can be taken before problems occur.

Universal Prompt Structures / Prompt Refinements
Definition

Universal prompt structures provide a clear template to explain processes step by step.

# Prompt

"Explain the working process of IoT-based predictive maintenance step by step."

# ChatGPT Output

Sensors are installed on machines.

Sensors collect operational data.

Data is transmitted to monitoring systems.

AI analyzes patterns and detects anomalies.

Alerts are sent to engineers for maintenance.

# Gemini Output

The process begins with installing sensors on equipment. Data is collected and analyzed continuously. If abnormal conditions are detected, maintenance alerts are generated.

Prompt Size Limitations
Definition

Large prompts are divided into smaller sections to make them easier for AI to process.

# Prompt

Large Prompt Example:
"Explain the entire IoT automation system used in a smart factory."

# Improved Prompt (Chunking):

Phase 1: Describe the hardware components of an IoT system.
Phase 2: Explain how data is collected and transmitted.
Phase 3: Describe how predictive maintenance alerts are generated.

# ChatGPT Output

Breaking large prompts into smaller parts allows AI to produce clearer and more structured explanations.

# Gemini Output

Chunking large prompts improves response quality and helps organize complex information step by step.

# Evaluation Method (Rubrics)

The responses from AI tools are evaluated based on the following criteria:

• Relevance – How well the response matches the prompt
• Clarity – How easy the response is to understand
• Actionability – Whether the response provides useful insights
• Specificity – Level of detail in the explanation
• Feasibility – Practical usefulness of the response
